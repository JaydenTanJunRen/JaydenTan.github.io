Featured Case Study

E-Vision — Smart Glasses for the Visually Impaired

On-device capture + AI description to give blind/low-vision users on-demand reading and scene understanding.

Problem

Visually impaired people struggle to read product labels and navigate unfamiliar spaces. Existing solutions are often phone-based and not always hands-free.

Solution

A clip-on glasses module (Pi Zero 2 W) with a camera at the bridge, three tactile buttons, and Bluetooth audio that:

  • Listens for user voice or button press
  • Captures an image on demand
  • Extracts text via OCR and reads it aloud
  • Sends scene images to a vision-capable GPT model for descriptive output

Technical Architecture

  • Wake-word detection: Porcupine (offline)
  • Speech-to-text: Whisper/whisper.cpp (on-device) or online API fallback
  • Intent parsing: GPT-powered JSON intent recognition
  • Vision/TTS: pytesseract for OCR; OpenAI Vision or custom model for scene descriptions; pyttsx3/espeak-ng for local TTS or cloud TTS for higher quality
  • Hardware: Pi Zero 2 W, Camera Module 3, Maker pHAT buttons, UPS Power HAT, Bluetooth earbud

Timeline & Milestones

  • Week 1: Core voice loop + capture + TTS proof-of-concept
  • Week 2: OCR integration + basic commands
  • Week 3: GPT intent parsing + voice polishing + user test

Key Learnings & Next Steps

  • Latency vs offline reliability trade-offs — build hybrid mode.
  • Add MEMS mic and custom PCB for form factor.
  • Conduct user interviews with visually impaired people for UX tuning.
